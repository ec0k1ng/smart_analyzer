# 自动化数据分析工具 当前状态

## 1. 当前阶段结论

项目当前处于“功能与文档已收口、自动化聚焦回归可通过、可进入发布构建前最后验收”的阶段。

当前必须以以下事实为准：

- 数据加载层已经具备 DAT、MDF、MF4 的两段式探测/映射/按需读取路径。
- GUI 与模板工作台已经完成模板编辑、目录化报告、曲线截图、导出格式选择和细节日志整理。
- HTML 与 Word 报告已经统一为 grouped_results 结构，并支持目录与曲线截图。
- Word 目录跳转问题已经通过安全短锚点和书签顺序修复收口。
- 当前剩余工作重点不是补功能，而是完成发布前构建与人工验收。

## 2. 当前已完成事实

### 2.1 分析闭环

- 已具备 CSV、Excel、DAT、MAT、MDF、MF4、BLF、ASC 输入能力。
- 已具备接口映射 Excel 驱动的标准化链路。
- 已具备派生量 -> KPI -> 规则结果 -> HTML/Word 报告导出的完整链路。
- 已具备 GUI 与 CLI 双入口。

### 2.2 当前数据加载层事实

- CSV、Excel、DAT、MDF、MF4 都支持探测实际列/通道后再按命中结果读取。
- DAT 会优先走测量文件解析，再回退到文本表格/INCA DAT 解析，并支持按列过滤读取。
- MDF/MF4 支持按命中通道过滤读取。
- BLF/ASC 总线解码已按 required_signals 预筛选，不再盲解全部 DBC 信号。
- BLF/ASC 的详细日志已经明确写出“按需 DBC 解码，不会展开全部总线信号”。

### 2.3 当前 GUI 能力

- 已支持文件入队、按 KPI 分组分析、结果查看、曲线查看、DBC 管理。
- 已支持派生量、KPI、模板、接口映射的内嵌编辑。
- 已支持运行日志清空、错误跳转、算法 print 输出收集。
- 已支持曲线页多面板、多信号拖拽、跨面板转移、横轴联动缩放。
- 已支持无分析结果时保留曲线信号配置并显示占位。
- 已支持模板编辑器的草稿创建、未保存标记、Ctrl+S 保存、Jinja 语法校验和打开模板目录。

### 2.4 当前报告能力

- HTML 报告支持左侧固定目录、分组/文件层级、目录收起/展开、正文居中不随目录移动。
- Word 报告首页支持可见目录，正文使用 Heading 1 / Heading 2 组织层级。
- Word 与 HTML 都支持每个数据项的曲线截图预览。
- Word 目录使用内部 hyperlink + bookmark，而不是 TOC 域占位符。

## 3. 当前已知风险

### 3.1 仍建议做一次真实客户端手工验收

- Word 目录跳转、HTML 目录布局和截图效果已经有自动化回归，但最终仍建议用真实 Word 与浏览器各打开一次新导出文件确认。
- 如果本次需要交付安装包，仍应在安装完成后做一次最小打开验证。

### 3.2 真实工程样本回归仍然偏少

- 当前仓库内仍缺少用户真实 BLF/MF4 + DBC 和真实 INCA DAT 的样本库。
- 这不影响当前发布候选状态，但会影响后续对现场兼容性的把握速度。
