TCS Smart Analyzer 当前需求总览

一、产品定位

本项目是一套本地离线运行的 TCS 打滑控制数据分析工具，面向工程师使用。它的目标不是做通用数据平台，而是围绕 TCS 这一类分析任务，提供从原始日志导入、接口映射、派生量与 KPI 计算、结果判定、曲线查看到 HTML/Word 报告导出的完整闭环。

二、当前用户需要理解的最小模型

当前用户只需要理解两件事：

1. KPI 或派生量定义里声明的 raw_inputs 名称
2. 接口映射表里手工填写的实际信号名

不再要求用户理解 signal_aliases、自动猜列、独立规则引擎等旧概念。

三、当前已经具备的主要能力

1. 支持 CSV、Excel、文本 DAT、MAT、MDF、MF4、BLF 数据输入
2. 支持 DBC 参与 BLF 与总线类 MDF/MF4 解码
3. 支持接口映射 Excel 表
4. 支持独立 Python 文件形式的派生量和 KPI 扩展
5. 支持 GUI 与 CLI 双入口
6. 支持曲线页多面板、多信号拖拽和横轴联动
7. 支持 HTML 与 Word 汇总报告导出
8. 支持 GUI 内嵌编辑 KPI、派生量、模板与接口映射

四、当前强约束

1. KPI 文件必须提供 calculate_kpi 和 calculate_kpi_series
2. KPI 的 trend_source 必须等于 KPI 自身 name
3. 派生量文件必须提供 description、algorithm_summary 和 CALIBRATION
4. 派生量通常应输出连续过程曲线
5. 如果派生量要表达全时段恒定值，必须先确认它是文件级固有标量属性，不能默认这样实现

五、当前状态说明

1. 当前项目功能闭环已经形成，可以继续迭代
2. 当前文档体系已经补齐到可交接状态
3. 当前完整测试套件不是全绿，后续需要先修测试基线再继续大规模功能开发
4. 当前仓库中已经没有 tcs_active_avg_slip_ratio 正式派生量文件，任何仍引用它的内容都应视为旧信息

六、当前接手建议

新接手者建议先读 README.md、docs/PROJECT_PLAYBOOK.md、docs/REQUIREMENTS.md、docs/PROJECT_STATUS.md、docs/HANDOFF.md、docs/ARCHITECTURE.md 和 docs/IMPLEMENTATION_LOGIC.md，再开始修改代码。
